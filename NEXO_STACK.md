# Nexo RH — Stack Produit V1 / V2

## Vision globale

Nexo est une plateforme RH operationnelle qui fluidifie le passage entre recrutement et onboarding.

Le produit transforme :

```text
besoin RH -> recrutement -> integration collaborateur
```

La force de Nexo n'est pas d'avoir toutes les fonctionnalites RH. La force de Nexo est de rendre fluide le passage entre un besoin de recrutement, un parcours candidat et un onboarding structure.

Phrase produit centrale :

```text
Nexo transforme une reunion RH en parcours candidat, puis en onboarding structure.
```

## Logique d'evolution

Nexo doit evoluer par couches.

- V1 : structurer le flux RH.
- V2 : automatiser les frictions observees sur le terrain.

Le passage V1 -> V2 ne doit pas etre une accumulation de fonctionnalites. Il doit venir des usages reels : ce que les clients font souvent, ce qui leur prend du temps, ce qui genere des oublis, ce qui merite d'etre automatise.

## V1 — Produit commercialisable rapidement

### Positionnement V1

```text
Transformez un besoin RH en parcours candidat puis onboarding.
```

### Objectif V1

Créer un flux RH simple et ultra fluide :

```text
Brief RH -> Pipeline candidat -> Onboarding
```

L'objectif business de la V1 est de chercher les premiers clients, les premiers vrais usages et les premiers retours terrain. L'objectif n'est pas encore de scaler.

### Fonctionnalites V1

#### 1. Brief RH

Le manager ou le RH remplit :

- intitule du poste ;
- besoin ;
- missions ;
- profil recherche ;
- urgence ;
- localisation ;
- type de contrat ;
- notes de reunion.

#### 2. Generation de poste

Nexo :

- structure le besoin ;
- genere une offre propre ;
- prepare le pipeline candidat.

#### 3. Gestion candidat

Pipeline simple :

- Nouveau ;
- Contacte ;
- Entretien ;
- Retenu ;
- Refuse.

Chaque candidat peut avoir :

- CV ;
- notes ;
- source ;
- commentaires ;
- historique.

Sources V1 possibles :

- LinkedIn ;
- site ;
- annonce ;
- cooptation ;
- autre.

#### 4. Onboarding

Quand un candidat est retenu :

- creation de la fiche collaborateur ;
- checklist d'arrivee ;
- documents demandes ;
- rappels ;
- suivi onboarding.

### Ce que V1 ne fait pas

- Pas d'API jobboards.
- Pas d'IA autonome complexe.
- Pas de multi-site avance.
- Pas de conformite RH avancee.
- Pas d'analytics lourds.
- Pas de scoring RH.
- Pas d'ATS enterprise.
- Pas de signature electronique.
- Pas de workflow ultra complexe.

### Pourquoi V1 est forte

La V1 resout deja :

- le chaos RH PME ;
- les recrutements desorganises ;
- les onboardings oublies ;
- la perte documentaire ;
- les suivis manuels.

### Cible ideale V1

PME et structures terrain :

- restauration ;
- retail ;
- evenementiel ;
- agences ;
- securite ;
- franchises ;
- artisans ;
- associations ;
- services.

## V2 — Automatisation RH intelligente

### Positionnement V2

La V1 structure. La V2 automatise.

La V2 doit arriver quand Nexo a :

- des clients ;
- des usages reels ;
- des donnees ;
- des retours terrain.

### Modules V2

#### 1. IA RH

Transformation de :

- reunion RH ;
- audio ;
- notes libres ;

en brief RH structure.

#### 2. Acquisition candidat

Tracking plus complet :

- LinkedIn ;
- jobboards ;
- site carriere ;
- cooptation ;
- campagnes.

#### 3. Publication d'offres

Diffusion multi-canal des offres.

#### 4. Dashboard RH avance

Suivi de :

- delais de recrutement ;
- taux de conversion ;
- sources performantes ;
- onboardings incomplets ;
- suivi des equipes.

#### 5. Automatisations

Exemple :

```text
candidat retenu -> onboarding cree automatiquement
```

Autres automatisations possibles :

- relances automatiques selon statut ;
- generation de checklist selon poste ;
- passage candidat vers collaborateur ;
- alertes sur documents manquants ;
- alertes manager.

#### 6. Permissions avancees

Roles :

- RH ;
- manager ;
- direction ;
- multi-entites.
